# Research Skill

## 用途
資料調查、產業分析、競品研究、技術評估、事實查核。

## 執行流程
1. 確認研究問題與範圍（從 Task Card 的 goal 取得）
2. 先查 project 內部資料（memory/、既有 outputs/）
3. 不足時再做 web search（最多 3 輪）
4. 整理成結構化筆記
5. 明確區分：已知事實 / 合理推論 / 待驗證 / 高風險假設
6. 輸出到指定位置

## 輸出格式
```markdown
# [研究主題]

## 結論
[一段話摘要]

## 已知事實
- ...

## 合理推論
- ...

## 待驗證
- ...

## 高風險假設
- ...

## 來源
- [來源1]
- [來源2]
```

## 品質標準
- 每個事實主張都要有來源
- 推論要標明推論依據
- 不確定的資訊標記 [待驗證]
- 不把搜尋結果直接複製貼上，要消化重組

## 常見失敗模式
- 搜尋結果不足就硬湊結論 → 應明確說「資料不足」
- 把單一來源當權威 → 應交叉比對
- 混淆事實與推論 → 應明確分類
